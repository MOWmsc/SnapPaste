# 2026-03-18

## Bug 修复：created_at 被重复复制覆盖

**问题**：`insertClip` 中检测到重复内容（hash 相同）时，会把 `created_at` 更新为当前时间，导致首次创建时间丢失。

**修复方案**：
- 新增 `last_pasted_at` 字段（`ClipRecord` 类型 + `database.ts`），用于记录最后一次从剪贴板检测到的时间
- `created_at` 不再被修改，永远保留首次创建时间
- 排序、过期清理、超限清理全部改为基于 `last_pasted_at`
- 列表项右侧简略时间改为展示 `last_pasted_at`，tooltip 中保留 `created_at` 作为"创建时间"
- 数据库加载时为旧记录补充 `last_pasted_at` 默认值（使用 `created_at`）

**涉及文件**：
- `src/types/index.ts` — ClipRecord 新增 `last_pasted_at` 字段
- `electron/database.ts` — insertClip、getClips 排序、cleanOldRecords、initDatabase 旧数据兼容
- `src/components/ClipboardItem.tsx` — 列表时间展示改用 `last_pasted_at`

## 窗口显示时自动滚动到顶部

**需求**：每次打开主页面时列表应回到最上方，而非保持上次的滚动位置。

**修改**：`src/components/ClipboardList.tsx`
- 添加 `VirtuosoHandle` ref 到 Virtuoso 虚拟列表组件
- 监听 `window-shown` 事件，触发时调用 `scrollToIndex({ index: 0, align: 'start' })` 滚到顶部
- 同时重置 `selectedIndex` 为 -1（取消选中状态）
