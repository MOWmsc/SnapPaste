## 复制后窗口隐藏速度优化
- 原问题：复制后先等 300ms toast 显示，再等 IPC invoke 往返，最后才隐藏窗口，体感明显延迟
- 优化方案：**先立即发送 hideWindow（fire-and-forget），再异步执行复制操作**
  - 非钉住模式：hideWindow 在 copyToClipboard IPC 之前发送，窗口几乎零延迟消失
  - 钉住模式：仍然显示 toast 反馈
- 移除了复制后的 300ms setTimeout 延迟

## 浮层交互 4 项 Bug 修复
1. **浮层消失太快无法滚动**：移除 `pointer-events: none`，改为 `pointer-events: auto`，浮层支持鼠标进入并滚动；鼠标离开条目后用 150ms 延迟隐藏，鼠标移入浮层时取消隐藏
2. **浮层出现两次（先下后上闪烁）**：移除 `transform: translateY(-100%)` 的条件判断，改为在 `setTooltipPos` 中一次性计算好 `above` 属性，使用 CSS 类 `.clip-tooltip-above` / `.clip-tooltip-below` 分别应用不同动画，避免 React 重渲染导致的位置跳动
3. **复制次数/时间不准确**：`copyToClipboard` 在 store 中调用后台 `recordCopy` 后，没有更新前端 state；修复为成功后同步更新 `set()` 中的 `copy_count`、`first_copied_at`、`last_copied_at`
4. **短文本不显示浮层**：移除 `text.length <= 80 && !text.includes('\n')` 的过滤条件，所有文本记录悬停都显示浮层（短文本仅显示统计面板，长文本额外显示完整内容区）
